# Cross-Artifact Consistency & Alignment Report

This optional command generates a consistency and alignment report across all project artifacts.

## Purpose
Ensure consistency and alignment between the constitution, specification, plan, and tasks before implementation.

## Instructions
When this command is invoked, you should:
1. Review all project artifacts:
   - Project constitution
   - Baseline specification
   - Implementation plan
   - Task list
2. Analyze for:
   - Consistency between artifacts
   - Alignment with project principles
   - Missing or conflicting requirements
   - Dependencies that are not properly tracked
   - Technical decisions that conflict
   - Gaps in the specification or plan
3. Generate a report identifying:
   - Inconsistencies found
   - Alignment issues
   - Recommendations for resolution
   - Areas of concern
4. Provide actionable recommendations to resolve issues

## When to Use
Run this command after `/speckit.tasks` and before `/speckit.implement` to ensure everything is aligned.

## Output
Generate a comprehensive consistency and alignment report with findings and recommendations.
